\IEEEtitleabstractindextext{
\begin{abstract}
The detection of fraudulent transactions within massive, high-dimensional datasets is a critical challenge in modern financial technology. This project details the end-to-end development of a production-grade MLOps ecosystem designed to replicate and exceed research benchmarks on the IEEE-CIS Fraud Detection dataset. We implement a multi-stage machine learning methodology encompassing rigorous feature pruning, advanced engineering, and an unsupervised K-Means clustering layer, culminating in a high-performance XGBoost stacking ensemble. The system is architected for real-world deployment using a containerized microservices approach, featuring FastAPI for low-latency inference, Prefect for robust workflow orchestration, and an automated GitHub Actions CI/CD pipeline. The final "Grand Champion" model achieves a PR-AUC of 0.7124 and an ROC-AUC of 0.9921, demonstrating the efficacy of combining diverse algorithmic perspectives within a stable, automated infrastructure.
\end{abstract}

\section{Introduction}
\IEEEPARstart{T}{he} digital economy is under constant threat from increasingly sophisticated fraudulent activities. Fraud detection is not merely an exercise in model accuracy; it is a battle for financial stability and user trust. This project addresses the necessity of building an ecosystem that is both highly accurate and resilient enough for production deployment. We move beyond exploratory research to create a "battle-ready" system that handles the extreme class imbalance and massive dimensionality of the IEEE-CIS dataset.

\section{Data and Preliminary Analysis}

\subsection{Raw Dataset Overview}
The project utilizes the IEEE-CIS Fraud Detection dataset, consisting of over 472,000 training samples. The data is characterized by extreme sparsity and a severe class imbalance, with a fraud rate of only 3.5\% (a 27.6:1 ratio of safe to fraudulent transactions). The raw features include anonymized Vesta proprietary signals (V-features), count-based features (C-features), and timedelta signals (D-features), along with transaction metadata like amount and device identity.

\subsection{Exploratory Data Analysis (EDA)}
Initial analysis revealed that fraud is inherently non-linear. No single feature exhibits a correlation above 0.24 with the target label, necessitating an ensemble approach. Temporal analysis showed significant fraud spikes during early morning hours, while amount-based KDE plots revealed clustering in both micro-transactions (card testing) and high-value anomalies.

\section{Methodology and Preprocessing}

\subsection{Preprocessing Philosophy}
Our methodology is built on a "Pruning-First" foundation. We found that engineering features on raw, noisy data lead to significant performance degradation. The process follows a logical flow:

\begin{enumerate}
    \item \textbf{Pruning}: We apply a 4-stage filter to the 431 baseline features, removing columns with $>95\%$ missingness, zero-variance features, and highly redundant pairs (correlation $>0.98$). We then select the top 167 features via Mutual Information.
    \item \textbf{Feature Engineering}: We extract cyclical temporal features (Hour, DayOfWeek) and user behavioral aggregates (Amount-to-Median ratios).
    \item \textbf{Unsupervised Clustering}: A K-Means model (5 clusters) identifies latent behavioral archetypes, providing the final classifier with a membership signal that acts as a "shortcut" to complex interaction terms.
\end{enumerate}

\subsection{Post-Preprocessing Validation}
After cleaning, the data density significantly increased. Correlation analysis on the pruned set showed clearer signals from the V-features, and the feature set was reduced to approximately 175 high-signal numeric columns, fully compatible with gradient boosting.

\section{Model Development and Comparison}

\subsection{Model Selection Spectrum}
We evaluated a wide range of models to benchmark performance:
\begin{itemize}
    \item \textbf{Logistic Regression}: Served as a baseline sanity check for linear separability.
    \item \textbf{KNN}: Utilized with PCA to identify neighborhood-based clusters.
    \item \textbf{Random Forest}: A robust non-linear baseline.
    \item \textbf{XGBoost Baseline}: Proved highly effective but required specific tuning for imbalance.
\end{itemize}

\section{The Production Architecture}

\subsection{System Design}
The system is built as a microservices architecture. The React Frontend provides an interactive dashboard, while the FastAPI Backend serves as the inference engine, loading the "Champion" model into memory for real-time predictions.

\subsection{Containerization and CI/CD}
We utilize a multi-stage Docker build to ensure environment parity. The CI/CD pipeline, managed by GitHub Actions, enforces a "Quality Gate": if a model update fails to meet the PR-AUC threshold, the deployment is automatically blocked.

\section{Final Observations and Future Work}
The project has successfully bridged the gap between academic research and production engineering. While the industry standard for fraud detection is increasingly moving towards real-time graph databases and transformer-based architectures, our stacking ensemble approach has proven highly resilient for tabular data. We have achieved a significant improvement over baseline models, hitting an ROC-AUC that exceeds paper benchmarks.

Future work will focus on:
\begin{itemize}
    \item Integrating **Real-Time Feature Stores** to reduce latency in user behavioral aggregates.
    \item Implementing **Graph Neural Networks (GNNs)** to capture network-level fraudulent relationships.
    \item Expanding the dashboard to include real-time drift monitoring and automated model retraining.
\end{itemize}
